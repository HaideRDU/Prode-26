Cloud Functions (recalculo de standings)
========================================

Nota Spark:
- Si el proyecto no usa plan Blaze, la app puede calcular clasificación del lado cliente
  leyendo `predictions`, `matches`, `tournamentResults` y `roomMembers`, sin depender
  de escritura en `standings/`.

- Despliegue: desde la raíz del proyecto, con Firebase CLI instalado y proyecto seleccionado:
  firebase deploy --only functions

- Emuladores locales:
  firebase emulators:start --only firestore,functions

- El cliente React NO puede escribir en `standings/` (ver firestore.rules). El recálculo ocurre en:
  - onPredictionWrite: al cambiar una predicción en `predictions/{predictionId}`
  - onMatchWrite: al cambiar un documento en `matches/{matchId}`
  - onTournamentResultWrite: al cambiar `tournamentResults/{questionId}`

- Para probar sin Functions: puedes ejecutar temporalmente reglas de desarrollo (no recomendado en producción)
  o desplegar solo las functions y usar datos reales en Firestore.

API-Sports (partidos en vivo)
=============================
- Requiere plan Blaze (Functions con red saliente + Cloud Scheduler).
- Liga Mundial 2026: league=1, season=2026 (API-Football v3 / API-Sports).
- Secreto (nunca en el repo). Opción A — script (lee APISPORTS_KEY de .env):
    npm run setup:apisports-firebase
  Opción B — manual:
    npx firebase login
    npx firebase functions:secrets:set APISPORTS_KEY
    npm run deploy:functions
- Tras seed de partidos, enlazar IDs de fixture:
    npm run link:api-sports
  (con APISPORTS_KEY y GOOGLE_APPLICATION_CREDENTIALS en .env)
- Prueba manual de sync en ventana horaria:
    npm run sync:api-sports-live
- Functions desplegadas:
    syncMatchesFromApiSports  → cada 5 min, solo entre 2026-06-10 y 2026-07-21 UTC
    linkMatchesApiSports      → callable autenticado para re-enlazar
- Ventana de polling: desde 15 min antes del partido hasta 3.5 h después (status live siempre).
- Al actualizar matches/, onMatchWrite recalcula standings automáticamente.
